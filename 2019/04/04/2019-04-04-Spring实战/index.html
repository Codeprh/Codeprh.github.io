第一章：spring之旅

spring核心知识

  • spring最根本的使命：简化java开发
  • 为了简化java开发，spring采取了下面4种关键策略
    1. 基于POJO的轻量级级和最小侵入性的编程
    2. 通过依赖注入和面向接口编程实现松耦合
    3. 基于切面和惯例进行声明式编程
    4. 通过切面和模板减少样板式代码
  • DI：能够让类之间保持松散耦合
  • AOP：把遍布应用各处的功能分离出来形成可重用的组件(关注点分离技术)
  • 使用模板消除样板式代码：jdbc和JdbcTemplate(spring)访问数据库

了解spring容器

  • 什么是spring容器：在spring应用中，对象由spring容器创建和装配，并存在容器之中。

  • spring容器种类：

    1. bean工厂(beanFactory)：最简单的容器，提供基本的DI支持。比较少用。
    2. 应用上下文（ApplicationContext）：基于BeanFactory创建，功能强大，经常使用的容器。

  • 常见应用上下文（类）：

    1. AnnotationConfigWebApplicationContext
    2. AnnotationConfigApplicationContext
    3. FileSystemXmlApplicationContext
    4. XmlWebApplicationContext
    5. ClassPathXmlApplicationContext

第二章：装配bean

  • spring装配的可选方案

    • 隐式的bean发现机制和自动装配(优先级最高)
    • 在java中进行显示配置(优先级中)
    • 在xml中进行显示配置(优先级低)

  • 自动化扫描和装配bean

    • spring从两个角度来实现
      1. 组件扫描(component scanning)：spring会自动发现应用上下文中的所创建的bean
      2. 自动装配(autowiring)：spring自动满足bean之间的依赖。
    • @component和@componetScan的关系
      • @componentScan是通过java方式显示启动组件扫描(默认不启动)，扫描所有带有@component的类。也可以通过xml的方式启动。扫描区间：配置类所在的包作为基础包来扫描组件
    • 使用@Autowired注解
      • @autowired是spring特有的注解，不管是构造器、Setter方法还是其他的方法，Spring都会尝试满足方法参数上所声明的依赖。假如有且只有一个bean匹配依赖需求的话，那么这个bean将会被装配进来。
      • 需要谨慎处理required=false属性和关于bean自动装配的歧义型的问题

  • @Autowired和@Inject、@Named注解

    • 二者都能实现组件自动装配，但是推荐使用spring特有的@autowired，
    • @Inject是java依赖注入规范，一般是配合@Named使用

  • 显示装配

    • javaConfig方式

      • 使用@Configuration是一个配置类，表明是通过javaConfig的方式来管理bean的创建和配置
      • @Bean：声明简单的bean，只需要在方法上面加上这个注解，return的对象就注册为spring上下文中的bean。bean的id：默认为方法名，可以通过name属性指定
      • 默认情况下：spring的bean都是单例的。

    • xml方式

      • 不推荐使用的方式，在spring刚出现的时候，xml是主要的方式。但是现在应该是使用自动化配置和javaConfig的方式。

什么是面向切面编程

  • Aop:横切关注点与业务逻辑相分离。横切关注点可以被模块化为特殊类，这些类被称为切面。
  • 常见 横切关注点：安全、事务、日志、缓存。
定义AOP术语

  • 常用术语：通知（Advice）、切点（pointcut）、连接点（joinpoint）
  • 通知：定义了切面是什么以及何时使用。定义了是什么、何时、
    • 5种类型的通知：
      • 前置通知(Before)
      • 后置通知(After)
      • 返回通知(After-returning)
      • 异常通知(After-throwing)
      • 环绕通知(Around)
  • 连接点：是在应用程序执行中能够插入的切面的一个点。
  • 切点：定义了何处，会匹配通知所要织入的一个或者多个连接点。缩小切面所通知的连接点的范围。
  • 切面：切面是通知和切点的组合。定义了：是什么、何时、何处。
  • 引入：向现有的类添加新方法或属性。
  • 织入：是把切面应用到目标对象并且创建新的代理对象的过程。
    • 织入时期（目标对象的生命周期）：
      • 编译期:需要特殊的编译器（AspectJ）
      • 类加载期：需要特殊的类加载器（ClassLoder）
      • 运行期：创建代理对象
Spring对AOP的支持
  • spring提供4种类型的AOP支持：（借鉴了AspectJ项目）
    • 基于代理的经典Spring Aop
    • 纯POJO切面
    • @AspectJ注解驱动的切面
    • 注入式AspectJ切面
  • Spring aop构建在动态代理基础之上，因此，spring对于aop的支持局限于方法拦截。
  • spring通知是java编写的。
  • spring在运行时通知对象。
  • spring只支持方法级别的连接点。

通过切点来选择连接点

  • spring aop中定义切点：要使用AspectJ的切点表达式语言来定义切点。

  • execution切点最常使用，并且它是实际执行匹配。而其他指示器都是用来限制匹配。

搭建Spring MVC

  • 配置DispatcherServlet，通过配置在web.xml文件中，但是有了Servlert3.0之后，我们可以使用Java将DispathcerServlet配置到Servlet容器中。

  • 自定义配置DispatcherServlet:

    • 扩展了org.springframework.web.servlet.support.AbstractAnnotationConfigDispatcherServletInitializer的任意类都会自动地配置DispatcherServlet和Spring应用上下文。
    • Spring的应用上下文会位于应用程序的Servlet上下文之中。
    • 配置了容器，才能讲上下文。
    • getServletMapping()：路径映射工作。
    • getServletConfigClasses()：返回带有@Configuration注解的类将会用来定义DispatcherServlet应用上下文中的bean。
    • getRootConfigClasses():返回带有@Configuration注解的类将会用来配置ContextLoderListener创建的应用上下文中的bean。

  • 两个应用上下文之间的故事

    • 两个上下文：DispatcherServlet会创建一个Spring应用上下文，另外一个是ContextLoderListener（Servlet监听器）创建的。
    • 通常，DispatcherServlet会加载：包含Web组件的bean，如控制器，视图解析器和处理器映射等。ContextLoderListener通常是驱动应用后端的中间层和数据层组件。
