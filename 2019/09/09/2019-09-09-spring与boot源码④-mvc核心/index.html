基础架构：Servlet

  • 特点
    • 请求/响应式（Request/Response）
    • 屏蔽网络通讯的细节
  • API特性
    • 面向HTTP协议
    • 完整生命周期
  • 职责
    • 处理请求
    • 资源管理（数据库连接、消息连接、其他）
    • 视图渲染

认识Spring MVC组件

组件Bean类型 说明
HandlerMapping 映射请求（Request）到处理器（Handler）加上其关联的拦截器（HandlerInterceptor）列表，其映射关系基于不同的HandlerMapping实现的一些标准细节。其中两种主要HandlerMapping实现，RequestMappingHandlerMapping支持标注@RequestMapping的方法，SimpleUrlHandlerMapping维护精确的URL路径和处理器的映射。
HandleAdapter 帮助DispatcherServlet调用请求处理器（Handle）,无需关注其中实际的调用细节。比如：调用注解实现的controller需要解析其关联的注解，HandlerAdapter的主要目的是为了屏蔽与DispatcherServler之间的诸多细节。
HandleExceptionRsolver 解析异常，可能策略是将异常处理映射到其他处理器、或者到某个HTML错误页面，或者其他
ViewResolver 从处理器（Handler）返回字符类型的逻辑视图名称解析出实际的view对象，该对象讲渲染后的内容输出到HTTP响应中。
LocaleResvoler 从客户端解析出Locale，为其实现国际化视图
MultipartResolver 解析多部分请求（如：web浏览器上传上文）的抽象实现

配置web mvc组件：app-context.xml

部署DispatcherServlet：web.xml

配置Spring MVC（注解驱动方式）

提供基于注解驱动的版本是spring framework3.1+之后提供的

  • 基本配置步骤

    • 注解配置：@Configuration（Spring模式(范式)注解）
    • 组件激活：EnableWebMvc(Spring模块装配)
    • 自定义组件：WebMvcConfigurer(Spring bean)

  • 实现

    这类替换了配置文件的：RequestMappingHandlerMapping、RequestMappingHandlerAdapter、InternalResourceViewResolver，还可实现自定义配置。

  • 注意事项：

    • 此时还是需要配置<context:component-scan base-package="/"/>和web.xml
    • 最终实现自动装配（去掉配置文件）：extends AbstractAnnotationConfigDispatcherServletInitializer

常见注解

  • 注册模型属性：@ModelAttribute
  • 读取请求头：@RequestHeader
  • 读取Cookie：CookieValue
  • 检验参数：@Valid、@Validated
  • 异常处理：@ExceptionHandler
  • 切面通知：@ControllerAdvice

配置Spring MVC（自动装配）

自动装配的起始版本是spring framework3.1+和Servler3.0之后提供的

Servler SPI

ServletContainerInitializerServlet规范接口，在Servlet容器启动之后，提供了一个回调方法javax.servlet.ServletContainerInitializer#onStartup来实现相应逻辑。

Spring适配Servler SPI

SpringServletContainerInitializer是spring实现Servlet规范接口（ServletContainerInitializer）的实现类。并且初始化Servler等工作交给@HandlesTypes(WebApplicationInitializer.class)来实现

Spring SPI

  • 基础接口：org.springframework.web.WebApplicationInitializer
  • 编程驱动：org.springframework.web.servlet.support.AbstractDispatcherServletInitializer
  • 注解驱动：org.springframework.web.servlet.support.AbstractAnnotationConfigDispatcherServletInitializer

实现

通过这种方式可以实现自动装配，也就是自动装配在spring就实现了，并且到这一步的时候才去掉了各种配置文件（xml）

配置Spring MVC（完全自动装配springboot）

  • 替换@EnableWebMvc:org.springframework.boot.autoconfigure.web.WebMvcAutoConfiguration
  • 自动装配DispatcherServlet:org.springframework.boot.autoconfigure.web.DispatcherServletAutoConfiguration

自动装配的顺序性

  • 绝对顺序：@AutoConfigureOrder
  • 相对顺序：@AutoConfigureAfter

条件装配

  • @ConditionalOnWebApplication(type = Type.SERVLET)
  • @ConditionalOnMissingBean(WebMvcConfigurationSupport.class)
