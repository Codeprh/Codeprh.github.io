Spring与Boot源码-理解SpringApplication

SpringApplication的使用

  1. 方式1

    1. SpringApplication.run(ApplicationConfiguration.class,args);
<!--￼0-->

  2. 方式3

    1. new SpringApplicationBuilder(NaohSpringBootApplication.class)
    .bannerMode(Banner.Mode.CONSOLE)
    .web(WebApplicationType.NONE)
    .profiles("prod")
    .headless(true)
    .run(args);
<!--￼1-->

配置SpringBoot Bean源

Java配置Class或者XML上下文配置文件集合（一个或者多个都行），用于SpringBootBeanDefinitionLoader读取，并且将配置源解析加载为Spring Bean定义。

  • Java配置Class
  • XML上下文配置文件

推断Web应用类型

根据当前应用应用ClassPath中是否存在相关实现来推断Web应用的类型，包括：

  • WebReactive：WebApplicationType.REACTIVE
  • WebServlet：WebApplicationType.SERVLET
  • 非Web：WebApplicationType.NONE

推断引导类（Main.Class）

依据Main线程执行堆栈判断实际的引导类

org.springframework.boot.SpringApplication#deduceMainApplicationClass

加载应用上下文初始器（ApplicationContextInitializer）

利用Spring工厂加载机制，实例化ApplicationContextIntitializer实现类，并排序对象集合。

org.springframework.boot.SpringApplication#getSpringFactoriesInstances(java.lang.Class, java.lang.Class<?>[], java.lang.Object…)

META-INF/spring.factories

实现步骤

  • 实现类：org.springframework.core.io.support.SpringFactoriesLoader
  • 配置资源：META-INF/spring.factories
  • 排序：AnnotationAwareOrderComparator

加载应用事件监听器（ApplicationListener）

和上面加载应用上下文初始器一样，利用spring工厂加载机制。

加载SpringApplication运行监听器（RunListeners）

spring工厂机制，读取SpringApplicationRunListener所有的实现类

运行SpringApplication运行监听器（RunListeners）

SpringApplicationRunListener、EventPublishingRunListener监听多个运行状态方法：

监听方法 阶段说明 SpringBoot事件 SpringBoot版本
starting spring应用刚启动 ApplicationStartingEvent 1.0
environmentPrepared ConfigurableEnvironment准备妥当，允许将其调整 ApplicationEnvironmentPreparedEvent 1.0
contextPrepared ConfigurableApplicationContext准备妥当，允许调整 1.0
contextLoaded ConfigurableApplicationContext已经装载，但仍未启动 ApplicationPreparedEvent 1.0
started ConfigurableApplicationContext已经启动，此时spring bean已初始化完成 ApplicationStartedEvent 2.0
running spring应用正在运行 ApplicationReadyEvent 2.0
failed spring应用运行失败 ApplicationFailedEvent 2.0

监听SpringBoot事件/spring事件

Spring Boot通过SpringApplicationRunListener的实现类org.springframework.boot.context.event.EventPublishingRunListener利用SpringFramerwork事件API监听。广播器（ApplicationEventMulticaster）Srping boot事件。

SpringFramework事件/监听器编程模型

  • spring应用事件
    • 普通应用事件：ApplicationEvent
    • 应用上下文事件：ApplicationContenxtEvent
  • Spring应用监听器
    • 接口编程模型：ApplicationListener
    • 注解编程模型：@EventListener
  • Spring应用事件广播器
    • 接口：org.springframework.context.event.ApplicationEventMulticaster
      • 执行模式：同步或者异步

创建spring应用上下文（ConfigurableApplicationContext）

根据在准备阶段推断的web应用类型来创建对应的上下文

上下文：

  • Web Servlet：org.springframework.boot.web.servlet.context.AnnotationConfigServletWebServerApplicationContext
  • Web Reactive：org.springframework.boot.web.reactive.context.AnnotationConfigReactiveWebServerApplicationContext
  • 非web：org.springframework.context.annotation.AnnotationConfigApplicationContext
