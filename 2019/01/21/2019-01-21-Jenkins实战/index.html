Jenkins 基本操作

  • 创建任务

    这里仅介绍Pipeline任务，MultiPipeline及其他任务类型不建议使用

    1. 登陆jenkins，点击新建任务，选择流水线任务类型
    2. 页面跳转至设置页，选择流水线添加相应的项目信息
    3. 选择构建触发器中的第三条创建一个gitlab推送触发器并打开高级设置生成一个调用secret token
    4. 完成任务创建

  • 关联Webhook给GitLab

    1. 登陆GitLab进入相关项目仓库，选择Setting - Intergrations 选项进入集成设置页面
    2. 复制jenkins任务中创建的触发器 url 和 secret token，创建webhook
    3. 测试验证

Pipeline 语法（声明式）

基本元素

  • pipeline

    所有有效的管道块必须包含在 pipeline 中

  • agent

    指定执行管道任务的目标节点

  • stages

    所有任务阶段的集合

  • steps

    任务实际内容

  • post

    结束动作

特殊声明

  • environment

    构建环境

  • options

    构建选项

  • when

    任务执行条件

  • 一般用不上的设置项

    • parameters 参数变量
    • triggers 任务触发设置
    • tools 指定构建工具环境
    • input 输入等待（不准备在公司环境用）

  • parallel

    在某个任务阶段中同时执行多个子任务
